2. Non-streaming flow complete()

Auth в middleware — единственный middleware. Sanitize и audit — внутри ChatService. Blocked запросы аудируются до raise.

3. Streaming flow complete_stream()

Токены идут клиенту сразу — настоящий TTFF. Output sanitizer убран полностью — к ответу LLM не применяется. Audit гарантирован через finally — при успехе, ошибке и дисконнекте.

4. SSE wire format OpenAI-compatible

Каждый чанк — одна строка data: {...}\n\n. Ошибки и успешные чанки — один и тот же формат потока. data: [DONE] гарантирован через finally — даже при ошибке.

5. AuditRecord v3.0+

messages и completion — None по умолчанию. Заполняются только при audit.body_logging.enabled: true. При streaming completion — сырой полный текст, собранный из буферизованных чанков.

6. Auth flow middleware

Единственный middleware. 401 возвращает до ChatService — не аудируется. Header: x-api-key (не Authorization: Bearer).
